# e-Rad / 競争的研究費 応募の落とし穴・書式ルール

府省共通研究開発管理システム（e-Rad）経由で研究費（JST・科研費・各種財団等）に応募する際の、**フォーム固有の制限・書式ルール・つまずきどころ**。e-Rad の応募フォームを Claude が代行入力・支援する前に読む。

制度ごとの個別値（費目・分野分類・字数上限・締切・予算枠）は各制度の**公募要領**＋応募管理リポの制度別ディレクトリを正とし、本書は**制度横断で効く e-Rad フォームの挙動**を扱う。出典 = JST 公募要領・e-Rad 操作マニュアル（例: JST 各事業の e-Rad マニュアル）＋ 実応募での観察。

> ⚠️ **本書の一般則は限られた制度での実応募観察に基づく**（全制度を網羅検証したものではない）。e-Rad は制度ごとに項目・制限・必須欄が異なるため、各記述は「観察された挙動」として扱い、実際の応募では必ず当該制度の公募要領・e-Rad マニュアルで裏取りすること。

## 応募フォームのタブ構成
- e-Rad の応募入力は **4 タブ構成**: **基本情報** / **研究経費・研究組織** / **個別項目** / **応募・受入状況**。各タブを順に埋めて保存する。
- 自由記述（研究目的・研究概要等）や課題名・分野は「基本情報」「個別項目」、金額・組織は「研究経費・研究組織」、他制度との重複・エフォートは「応募・受入状況」に分かれる（各タブの詳細は本書の該当節を参照）。

## 応募情報ファイル（提案書の添付）
- e-Rad の「**応募情報ファイル**」欄 = 応募者が作成した提案様式（提案書）の **PDF をアップロードする欄**（「参照」→ ファイル選択 →「アップロード」）。e-Rad の Web 入力情報と結合され、1 つの応募内容提案書 PDF になる。
- 1 ファイル・最大容量は制度規定（30MB / 10MB 等、要項確認）。
- **提出形式は制度依存**: 応募情報ファイルとして **PDF をアップロードする制度**（JST 等）と、**様式の xlsx/docx 本体をそのまま提出する制度**（科研費系）がある。要項で指定形式を確認する（様式の fill → PDF 化の手順は [office-automation.md](office-automation.md)）。
- 提出するのは **完成・承認済みの正式版 PDF**。生成途中の下書き・中間レンダリング・編集用 docx と取り違えない。

## 文字種の制限
e-Rad の入力欄は厳格な charset 制限があり、**欄の種類で置換戦略が違う**:
- **研究課題名（英語）など半角欄**: 半角（ASCII）限定。ギリシャ文字（μ, τ, α …）・全角マイナス（−, U+2212）・全角記号はエラー（「半角文字で入力してください」）。→ **英語綴り / ASCII に置換**（μ→mu, τ→tau, −→半角ハイフン）。カタカナ置換は英語欄では不適。
- **研究目的・研究概要・役割分担等の日本語 long-textarea**: ギリシャ文字・上付下付・数学記号・`§`・エム・ダッシュ・丸付き数字・ローマ数字・機種依存文字を弾く。→ **カナ / 日本語に置換**（σ→シグマ、①→(1)、—→、）。日本語欄でも全角は可だがこれら弾かれ文字は除く。
- **禁止文字の詳細リスト（置換表）は [office-automation.md #erad-forbidden-chars](office-automation.md#erad-forbidden-chars) が正本**。応募様式（xlsx/docx）の draft 段階で排除しておくと、様式と e-Rad textarea の二重 maintenance が不要。

## 字数制限（フォーム依存）
- 研究目的・研究概要・役割分担・相違点 等の自由記述欄には **字数上限**がある（**改行・スペースも 1 文字としてカウント**することが多い）。上限は制度・欄ごとに異なる。
- 超過すると **末尾が黙って切れる** か保存時エラーになる。→ **貼り付け前に文字数を実測**し、貼付後にフォームのカウンタも確認する。コピペで末尾に改行・空白が混入して超過することがあるので末尾を点検。
- 制度によっては自由記述欄に「提案書参照」と書かせる運用もあるが、実内容の記入を求める制度もある（要項に従う）。

## 研究分野は二系統（混同注意）
- **制度独自の大分類**（ラジオボタン等の固定リスト）= 提案様式の分野欄と一致させる。
- **e-Rad 共通の研究分野（「研究の内容検索」ポップアップ）** = 科研費系の細目分類。制度の大分類名で検索しても実体に合う細目は出ないことが多い。**研究の実体**を表すキーワード（例: 理論素粒子なら「素粒子」→「素粒子・原子核・宇宙線・宇宙物理」）で検索・選択する。両者は独立した別項目。

## 研究経費・研究組織
- 金額は **千円単位**で入力する欄が多い（"____,000 円"）。
- **間接経費 = 直接経費の一定率**（10% 等、制度規定）。固定で自動計算 or 手入力。様式が消費税欄空欄なら 0。
- **「研究組織」の各メンバー直接経費は初年度分のみ**（全期間総額ではない）。「申請額（初年度）」と「研究者ごとの金額合計」が一致して差額 0 になればよい。
- **直接経費と間接経費は別欄**。合計を 1 欄に入れない（合計は自動表示）。
- **年度配分**: 日本の年度 = 4月〜翌3月。研究期間・招へい/滞在期間が年度境界（3月→4月）を跨ぐ費目は年度分割（例: 1〜6月の活動 → 1〜3月＝前年度・4〜6月＝当年度に按分）。1 年度に収まれば全額その年度・他年度 0（必須欄なので 0 を明示入力）。

## 応募・受入状況（不合理な重複・過度の集中）
- **「研究内容の相違点及び他の研究費に加えて本応募研究課題に応募する理由」欄を、応募中・実施中の他の競争的資金すべてに記入**する。本応募自身の行は空欄。テーマ・対象・手法・経費使途の違い＋当該資金が固有に必要な理由を簡潔に書く。
- 同制度の別申請を先に提出済みだと、それも「応募中」として並ぶ。同制度同士は **招へい者・テーマ等の独立性**を明記。
- **エフォート欄は応募者自身の判断値**で、提案書様式側に規定値（推奨%）はない。応募者が実態に即して決める。
- **「その他の活動」のエフォートで全体 100%**。応募中課題のエフォートは応募時点では他活動に計上されない（= 採択までは加算されず、採択時に実態へ反映して再調整。実施中＋その他の活動で 100%）。教育 0% 等の不自然な配分は審査で目立つので実態に合わせる（teaching を 0% にすると査読印象が悪い）。
- 学位欄の「(Unregistered)」= e-Rad 研究者情報に学位の取得年月日・大学が未登録の表示。多くの場合チェックのみで申請は通る（直すなら研究者情報を別途編集）。

## 機関承認・提出フロー
- e-Rad 上の **所属研究機関承認の要否は制度による**（研究者提出だけで完結する制度もある）。ただし研究者招へい・国際協定等を伴う場合、**学内の機関決裁（メール経由）が先に必要**なことが多い。
- 一般的フロー: 申請書を **研究支援事務へメール提出** → **機関決裁（学長承認等）** → 承認連絡後に **研究者が e-Rad で提出**。
- **機関決裁は時間を要する**。配分機関の締切の **2 週間前まで**に事務へ相談・提出するのが安全（締切直前は e-Rad 混雑で提出処理が間に合わないリスクもある）。
- **課題 ID は応募作成時に e-Rad が自動採番**（手入力不可）。問い合わせ時に使うので控える。

## 提出完了の判定・提出後
- 締切時刻までに応募ステータスが **「配分機関処理中」または「申請中／応募中」**になっていれば正常完了（「受理」は必須ではない）。なっていなければ無効。
- 「提出済の課題 ＞ 課題一覧」でステータスを確認。控え（応募内容提案書 PDF）は「応募内容提案書のプレビュー」or 課題一覧から取得できる。
- 機関決裁した事務へ **提出完了の一報**を入れると事務が状態を把握できる。
