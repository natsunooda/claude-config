# 物理・数理ノート規約

物理・数理の導出ノート (LaTeX/PDF/md、共同研究の note・解説・手検算 reference) を書く・改稿するときの規約。2 つのルールはどちらも「後から取り違える経路を構造的に塞ぐ」ためのもの。

## 1. 脚 (添字) は常に全部顕に書く

**ルール:** 数式で添字を省略しない。inline shorthand (`hG⁽¹⁾h`、`C⁽¹⁾C⁽¹⁾`、`êΓê`、`δ²S/δêδê` の類) を書かず、縮約と functional derivative は添字を全部書く:

- `h^{µν} G⁽¹⁾_{µν}`、`C⁽¹⁾_{µνρσ} C⁽¹⁾^{µνρσ}`、`ê_{bν} Γ⁽²⁾_{(bν)(cρ)} ê_{cρ}`、`δ²S/(δê_{bν} δê_{cρ})`
- worked example・boxed 主結果・付録も例外なし
- 表示数式 (display) を使う。inline に押し込むと省略する圧力が生まれる

**Why:** 省略した脚を後から「復活」させるときに、反対称化重み (1/3!、2!)・対称化の ½・mode の規格化・Taylor の ½ (Γ = δ²S) と field rescale の組合せを取り違えるのが定番事故。kernel-level と action-level の係数取り違えも省略簿記から生じる。読者 (共著者・未来の自分) の追計算の最大障壁も省略形の再構成コスト。

**How:**
- 数式を書いた直後に「省略された縮約・脚はないか」を 1 周 scan し、あれば display で書き下す
- 組合せ重み (1/k!、射影子ノルム、½) が絡む等式は、数値検証 (numpy 数行) を 1 本添えて regression 化する

**事例 (一般化):** 添字を省略した quadratic form から係数を読んだノートで、5 つの係数が単一の規約で読めない混在 (項により ×4 / ×2 のズレ) になっていた。全添字化にともなう機械検証で、「この mode は他の射影子に消される」と書いた前提が実は二次形式直交でしか成立しない箇所 (off-diagonal 射影子は operator としては消さない) も発見された。省略形のままでは両方とも見えない。

## 2. ノートは snapshot、歴史は repo 側 (md + git)

**ルール:** 配布物としてのノート (tex/PDF) には「その時点の最良の知見」だけを self-contained に書く。旧版の誤り・訂正経緯・version 史話 (「v1 ではこうだった」「旧 narrative を訂正」) はノート本文に入れない。履歴の home は repo 側の md (SESSION/DESIGN/RETRACTIONS 等) と git log。

これは CONVENTIONS.md §2 の **snapshot 原理** (dynamic docs は現状のみを記録し、履歴は git log に委ねる) の deliverable 文書への拡張。

**Why:**
- ノート内の史話は、読者に旧誤りを再 import させる毒餌になる (誤った係数や撤回済み narrative がノートから引用され続ける)
- self-contained 性が壊れる (史話は外部文脈を前提にする)
- 未配布の版に対する version 注記は読者にとって無意味な認知負荷

**How:**
- 改稿時に各段落へ「これは現在の最良知見か、経緯の説明か」を問い、経緯は md へ移してノートから削除する
- supersession marker・新旧換算表は md 側に置き、ノートからは参照しない (ノートは単体で完結)
- scripts や他の docs からノートを参照するときは、式番号・節番号でなく**安定な §名/topic 名**で参照する (ノートは進化して番号がズレる)
- 例外: 公開後 (arXiv 等) の正誤訂正は erratum の標準作法に従う (この規約は未公開の作業ノートが対象)
